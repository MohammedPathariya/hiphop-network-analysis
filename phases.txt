New Data Collection: Write a robust script that loops through every artist and every song in your network and scrapes the lyrics for all of them.

Phase 4: Content Analysis (NLP)

Phase 1: Data Collection (The Harvest) üï∑Ô∏è
Goal: Build the raw dataset. This is the most fragile step because it relies on an external API and web scraping. Notebook: 01_Data_Collection.ipynb

Define Seed Artists: Pick 3-4 diverse "pillars" of hip-hop to ensure distinct communities.

Suggestion: Kendrick Lamar (Lyricism/West Coast), Future (Trap/Atlanta), J. Cole (Storytelling/East Coast), Drake (Pop-Rap/Global Bridge).

The Crawl Logic:

For each Seed Artist, get their top ~50 songs.

For each song, extract: Title, Release Date, Featured Artists, and Lyrics.

Recursive Step (The Network Expander): Collect the names of all featured artists. If you have time, get the top 5 songs for those featured artists too (Depth 1.5). Warning: This grows exponentially. Start small.

Error Handling: Implement try/except blocks and time.sleep(1) between requests so Genius doesn't ban you.

Deliverable: Two files in data/processed/:

network_edges.csv: Columns [Source_Artist, Target_Artist, Song_Title]

song_lyrics.csv: Columns [Song_ID, Artist, Title, Lyrics]

Phase 2: Network Analysis (The Skeleton) üï∏Ô∏è
Goal: Map the social structure and identify the "scenes" (communities). Notebook: 02_Network_Analysis.ipynb

Graph Construction:

Load network_edges.csv.

Build a graph using NetworkX where Nodes = Artists, Edges = Collaborations.

Centrality Analysis:

Calculate PageRank (Who is the "main character" of the network?).

Calculate Betweenness (Who bridges the gap between Kendrick and Future?).

Community Detection (Crucial):

Run the Louvain Algorithm.

Assign every artist to a Community_ID.

Check: Do the communities make sense? (e.g., Does Community 1 look like "Atlanta Trap" and Community 2 look like "Conscious Rap"?).

Deliverable:

A clean Network Graph visualization (filtered ego network).

Phase 3: NLP & Topic Modeling (The Flesh) üó£Ô∏è
Goal: Analyze the substance of the lyrics using AI. Notebook: 03_NLP_Topic_Modeling.ipynb

Preprocessing (The Grind):

Load song_lyrics.csv.

Clean: Remove bracketed text like [Verse 1], [Chorus].

Tokenize: Split text into words.

Stopwords: Remove common words (the, yeah, like, got) and slang that appears everywhere (e.g., ain't might be a stopword if it doesn't help distinguish topics).

Topic Modeling (LDA):

Use Gensim or SciKit-Learn to run Latent Dirichlet Allocation (LDA).

Hyperparameter Tuning: Try asking for 5 topics, then 10, then 15. Which number gives the most distinct, human-understandable topics?

Topic Labeling:

Look at the top words for "Topic 1". If they are money, cars, benz, label it "Materialism/Flexing".

If "Topic 2" is love, baby, girl, label it "Romance/R&B".

Deliverable: A dataframe song_topics.csv where every song has a % score for each topic. (e.g., "Money Trees" = 60% Struggle, 40% Flexing).
